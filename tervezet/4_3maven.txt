Egy projekt menedzselésére és a build folyamatok automatizálására egy szoftver.
Menedzseli a dependency-ket: pom.xml
pom.xml: egy egyszerű xml de ez tartalmazza a legtöbb információt a maven számára ő ebből dolgozik.
Összeállítják a packaget: jar vagy war kiterjesztés, project név, artifactid, version Lefordíttatja a kódot a java compilerrel stb.

CENTRAL REPOSITORY: mavennek egy repository ahova már feltöltöttek kül szoftvercsomagokból kiadásokat(kódokat). Ezeket le lehet húzni a projektünkbe.

<parent>-> milyen magot szeretnénk használni (org.springframework.boot spring boot starter parent) a spring bootból ez már rengeteg dolgot beinjektál magába
mi a projektünkben kiterjesztünk egy meglévő kódállományt

A build folyamat során fázisok vannak. Egy fázist meghívva az előtte levő fázisok sorra lefutnak addig a pontig.
Például:
-mvc compile
    validate
    generate-sources
    process-sources
    generate-resources
    process-resources
    compile

Maven Phases(A legfontosabb fázisok)
- mvn
    validate: validálja a projektet és ellenőrzi, hogy minden információ a rendelkezésre áll-e	
    compile: Lefordítja a forráskódot
    test: teszteli a lefordított kódot a rendelkezésre álló unit tesztelő keretrendszerrel.
    package: Fogja a lefordított kódot és a megadott kiterjesztésű csomagot(package) elkészíti.
    integration-test: Futtatja illetve kitelepíti a csomag(package) ha szükséges egy olyan környezetbe ahol az integrációs tesztek futtathatóak.(modulok összekapcsolva)
    verify: Bármilyen ellenőrzés futtatása amivel a csomag(package) érvényessége ellenőrizhető és megfelel-e az elvárt irányelveknek.
    install: A csomag(package) telepítése a helyi tároló rendszerbe(repository), hogy az használható legyen függőségként(dependency) más helyi projekt számára.
    deploy: Felmásolja a végleges csomagot(package) egy távoli tároló rendszerbe(repository), hogy megosztható legyen más fejlesztőkkel és projektekkel.
	

-mvn install -> leszedi a repository-ból a dependency-ket vagyis a szükséges szoftverkomponenseket.
		létrehoz egy target mappát és oda fordíttatja le a java compilerrel a java-t classra(bítekód)
		majd összeállítja a jar vagy war-t.
-mvn clean -> törli a targetből a dolgokat	
-mvn site-> pom ból generál egy információs file-t a site-ról
